# Challenge: Learning Hackathon — Spec Driven Development

## Overview
- **Platform:** Devpost
- **URL:** https://learn-ai.devpost.com/
- **Deadline:** April 29, 2026 @ 5:00pm EDT
- **Prize:** Digital completion badge + possible platform feature callout (no cash prizes)
- **Format:** Online, solo or team, ~4-6 hours of active work

## CRITICAL: This is completion-based, not competitive
There are no judges ranking submissions. You either complete the full workflow or you don't.
The only real competition is for the "standout project" platform feature callout.

## Hard Requirements
1. Claude Pro or Max subscription
2. Git + terminal + desktop computer (no mobile)
3. Install the `devpost-curriculum` plugin via Claude Code marketplace
4. Complete all 8 commands in order: /onboard → /scope → /prd → /spec → /checklist → /build → /iterate → /reflect
5. Submit: project description + lessons learned write-up
6. Submit: zipped `docs/` folder (scope doc, requirements doc, technical spec, build checklist)
7. Submit: public repo link with source code and working run instructions

## Judging (completion-based)
All criteria are pass/fail completion. Quality signals that win the feature callout:
- Tight, explicit scope document (clear in/out boundaries)
- Professional user stories with acceptance criteria in the requirements doc
- Real architecture decisions in the tech spec (not hand-waving)
- Granular build checklist (independently verifiable steps)
- Working deployed app (live URL >> local only)
- Genuine, specific /reflect output (not generic "planning is good" platitudes)

## What Most Participants Miss
- Treating the docs as a chore — the docs ARE the submission reviewers read first
- Skipping /iterate — signals lack of thoroughness
- No live deployment URL — local-only submissions lose credibility
- Generic lessons learned text — easiest way to not get featured
- Scope creep during /build — breaks the integrity of the spec-driven story

## Key Tactical Notes
- Do the entire 8-command pipeline in ONE uninterrupted 4-6 hour session
- Speech-to-text (e.g. Superwhisper) is officially recommended to speed up interview phases
- Submit at least 2 weeks early — no benefit to waiting until Apr 29
- Simple app + excellent docs beats complex app + rushed docs every time
